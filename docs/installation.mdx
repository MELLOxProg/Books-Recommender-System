## Installation Methods

### Method 1: Quick Setup (Recommended)

<Steps>
  <Step title="Clone Repository">
    ```bash path=null start=null
    git clone https://github.com/MELLOxProg/Books-Recommender-System.git
    cd Books-Recommender-System
    ```
  </Step>
  
  <Step title="Create Virtual Environment">
    <Tabs>
      <Tab title="Windows (Command Prompt)">
        ```cmd path=null start=null
        python -m venv book-recommender
        book-recommender\Scripts\activate
        ```
      </Tab>
      
      <Tab title="Windows (PowerShell)">
        ```powershell path=null start=null
        python -m venv book-recommender
        .\book-recommender\Scripts\Activate.ps1
        ```
      </Tab>
      
      <Tab title="macOS/Linux">
        ```bash path=null start=null
        python3 -m venv book-recommender
        source book-recommender/bin/activate
        ```
      </Tab>
    </Tabs>
  </Step>
  
  <Step title="Install Dependencies">
    ```bash path=null start=null
    # Upgrade pip first
    pip install --upgrade pip
    
    # Install project dependencies
    pip install -r requirements.txt
    ```
  </Step>
  
  <Step title="Verify Installation">
    ```bash path=null start=null
    # Test basic imports
    python -c "import streamlit, numpy; print('âœ… Dependencies installed successfully!')"
    
    # Check model files
    ls artifacts/
    ```
  </Step>
</Steps>

### Method 2: Development Setup

For contributors and developers who want to modify the system:

<Steps>
  <Step title="Fork and Clone">
    1. Fork the repository on GitHub
    2. Clone your fork:
    ```bash path=null start=null
    git clone https://github.com/YOUR-USERNAME/Books-Recommender-System.git
    cd Books-Recommender-System
    ```
  </Step>
  
  <Step title="Development Environment">
    ```bash path=null start=null
    # Create development environment
    python -m venv book-recommender-dev
    
    # Activate environment
    source book-recommender-dev/bin/activate  # macOS/Linux
    # or
    book-recommender-dev\Scripts\activate     # Windows
    ```
  </Step>
  
  <Step title="Install in Development Mode">
    ```bash path=null start=null
    # Install with development dependencies
    pip install -e .
    pip install jupyter matplotlib seaborn
    
    # Install pre-commit hooks (optional)
    pip install pre-commit
    pre-commit install
    ```
  </Step>
  
  <Step title="Run Development Server">
    ```bash path=null start=null
    # Run with auto-reload
    streamlit run app.py --server.runOnSave=true
    ```
  </Step>
</Steps>

### Method 3: Docker Installation

For containerized deployment:

<Steps>
  <Step title="Install Docker">
    - **Windows/macOS**: [Docker Desktop](https://www.docker.com/products/docker-desktop)
    - **Linux**: [Docker Engine](https://docs.docker.com/engine/install/)
  </Step>
  
  <Step title="Create Dockerfile">
    ```dockerfile path=null start=null
    FROM python:3.9-slim
    
    WORKDIR /app
    
    COPY requirements.txt .
    RUN pip install --no-cache-dir -r requirements.txt
    
    COPY . .
    
    EXPOSE 8501
    
    CMD ["streamlit", "run", "app.py", "--server.address", "0.0.0.0"]
    ```
  </Step>
  
  <Step title="Build and Run">
    ```bash path=null start=null
    # Build image
    docker build -t book-recommender .
    
    # Run container
    docker run -p 8501:8501 book-recommender
    ```
  </Step>
</Steps>

If the repository includes pre-trained models:

## Platform-Specific Instructions

<Tabs>
  <Tab title="Windows">
    ### Windows-Specific Setup
    
    **Using Command Prompt:**
    ```cmd path=null start=null
    # Clone repository
    git clone https://github.com/MELLOxProg/Books-Recommender-System.git
    cd Books-Recommender-System
    
    # Create virtual environment
    python -m venv venv
    venv\Scripts\activate
    
    # Install dependencies
    pip install -r requirements.txt
    
    # Run application
    streamlit run app.py
    ```
    
    **Common Windows Issues:**
    - **PowerShell Execution Policy**: Run `Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser`
    - **Long Path Support**: Enable in Windows settings or use `\\?\` prefix for paths
    - **Antivirus**: Add project folder to exclusions if needed
  </Tab>
  
  <Tab title="macOS">
    ### macOS-Specific Setup
    
    **Using Homebrew (Recommended):**
    ```bash path=null start=null
    # Install Python via Homebrew
    brew install python
    
    # Clone and setup
    git clone https://github.com/MELLOxProg/Books-Recommender-System.git
    cd Books-Recommender-System
    
    # Create virtual environment
    python3 -m venv venv
    source venv/bin/activate
    
    # Install dependencies
    pip install -r requirements.txt
    
    # Run application
    streamlit run app.py
    ```
    
    **macOS Considerations:**
    - Use `python3` and `pip3` commands explicitly
    - May need Xcode command line tools: `xcode-select --install`
    - For M1/M2 Macs, ensure all packages have ARM64 support
  </Tab>
  
  <Tab title="Linux">

## Troubleshooting Installation Issues

<AccordionGroup>
  <Accordion title="SSL Certificate Errors">
    **Problem**: SSL errors when downloading packages
    
    **Solutions**:
    ```bash path=null start=null
    # Option 1: Upgrade certificates
    pip install --upgrade certifi
    
    # Option 2: Use trusted hosts
    pip install -r requirements.txt --trusted-host pypi.org --trusted-host pypi.python.org
    
    # Option 3: Corporate networks
    pip install -r requirements.txt --proxy http://your-proxy:port
    ```
  </Accordion>
  
  <Accordion title="Permission Denied Errors">
    **Problem**: Permission issues during installation
    
    **Solutions**:
    ```bash path=null start=null
    # Linux/macOS - avoid sudo, use --user flag
    pip install --user -r requirements.txt
    
    # Windows - run as administrator or check folder permissions
    icacls "C:\path\to\project" /grant Everyone:F /T
    ```
  </Accordion>
  
  <Accordion title="Package Conflicts">
    **Problem**: Dependency version conflicts
    
    **Solutions**:
    ```bash path=null start=null
    # Create fresh environment
    python -m venv fresh-env
    source fresh-env/bin/activate  # or activate.bat on Windows
    
    # Install specific versions
    pip install streamlit==1.28.0 numpy==1.24.0
    
    # Check for conflicts
    pip check
    ```
  </Accordion>
  
  <Accordion title="Large File Download Issues">
    **Problem**: Model files or data downloads fail
    
    **Solutions**:
    ```bash path=null start=null
    # Use Git LFS for large files
    git lfs install
    git lfs pull
    
    # Alternative: Manual download
    wget https://raw.githubusercontent.com/MELLOxProg/Books-Recommender-System/main/artifacts/model.pkl
    
    # Verify file integrity
    python -c "import pickle; pickle.load(open('artifacts/model.pkl', 'rb')); print('âœ… File valid')"
    ```
  </Accordion>
</AccordionGroup>
